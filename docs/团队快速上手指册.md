# AI Coding Harness 团队快速上手指册

> 本手册面向业务仓库开发团队成员。
> 你只需要做两件事：① 把规范源库接入你的业务仓库；② 用脚本生成提示词，复制给 AI 工具执行。
> 完整流程规范、方案对比和验收标准详见 [docs/业务仓库接入与执行SOP.md](业务仓库接入与执行SOP.md)。

---

## 第一步：接入规范源库

在业务仓库根目录执行一条命令，完成所有初始化。

### 1.1 前置条件

- 业务仓库已初始化 git（根目录有 `.git`）
- 本机已 clone 规范源库（脚本需要读取本地文件来运行 bootstrap）

```bash
# 先 clone 规范源库到你的工作区（只需一次）
cd ~/workspace
git clone https://github.com/chengqiZ/harness-engineering.git
```

### 1.2 执行初始化

```bash
# 进入业务仓库根目录
cd ~/workspace/my-business-repo

# 执行接入脚本，参数是规范源库的 GitHub 地址
bash ~/workspace/harness-engineering/scripts/init_business_repo.sh https://github.com/chengqiZ/harness-engineering.git
```

### 1.3 验证接入成功

```bash
./ai status
```

看到以下输出即表示接入成功：

```
repo: my-business-repo
branch: main
standards: .ai-harness/.ai-standards
specs:
```

### 1.4 提交初始化结果

```bash
git add .gitmodules .ai-harness .github ai
git commit -m "chore(harness): 接入 AI Coding Harness"
```

接入后业务仓库会新增以下内容：

| 路径 | 说明 |
|---|---|
| `.ai-harness/.ai-standards/` | 规范源库 submodule（锁定版本，只读） |
| `.ai-harness/AGENTS.md` | 项目特例文件（你只写本项目与标准不同的地方） |
| `.ai-harness/specs/` | 需求 spec 目录（每个需求一个子目录） |
| `.ai-harness/docs/changelog/` | 变更记录 |
| `.github/pull_request_template.md` | PR 模板 |
| `./ai` | 日常命令入口（Codex 专用） |

## 第二步：用脚本生成提示词，复制给 AI 工具

> **核心思路**：你不需要自己写提示词，脚本会根据你的需求参数自动填充模板，生成一段完整的、可以直接粘贴给 AI 的指令。

### 2.1 整体流程图

```
需求文档 → [脚本生成提示词] → 复制给 AI → AI 读规范+读需求 → 产出代码/spec
```

每个阶段都有对应的脚本：

| 阶段 | 脚本 | 生成什么提示词 | 给 AI 后 AI 做什么 |
|---|---|---|---|
| 生成 spec | `prepare_spec_prompt.sh` | 需求分析提示词 | AI 读需求文档，输出 spec 四件套 |
| 开发 task | `prepare_task_prompt.sh` | 单任务开发提示词 | AI 只实现指定 task，输出代码+测试 |
| 生成 PR | `prepare_pr_prompt.sh` | PR 准备提示词 | AI 输出 PR 描述、风险、回滚方案 |
| 串行全流程 | `prepare_serial_task_prompt.sh` | 串行执行提示词 | AI 按 task 依赖顺序一个一个执行 |

### 2.2 阶段一：新需求 → 生成 spec

当你拿到一个新需求文档，先让 AI 帮你把它变成规范化的 spec 四件套。

**准备需求文档**（任意格式，PRD、方案说明、缺陷描述都可以）：

```bash
mkdir -p docs/requirements
# 把需求文档放到这里，例如 docs/requirements/login-page.md
```

**运行脚本生成提示词**：

```bash
bash .ai-harness/.ai-standards/scripts/prepare_spec_prompt.sh login-page docs/requirements/login-page.md
```

**复制输出给 AI 工具**（Claude Code、Cursor、Gemini CLI 等），AI 会：

1. 读取需求文档 + 规范
2. 输出四个文件到 `.ai-harness/specs/login-page/`：
   - `01-requirements.md` — 需求梳理
   - `02-design.md` — 设计方案
   - `03-tasks.md` — 任务拆解（含 Task ID、复杂度、依赖）
   - `04-acceptance.md` — 验收标准
3. 告你推荐先做哪个 task、有没有缺失信息

**你确认**：检查 spec 四件套，确认需求和任务拆解合理后，再进入开发。

### 2.3 阶段二：开发单个 task

确认 spec 后，让 AI 只实现一个 task。

**查看有哪些 task**：

```bash
cat .ai-harness/specs/login-page/03-tasks.md
```

**运行脚本生成提示词**（假设第一个 task 是 T1）：

```bash
bash .ai-harness/.ai-standards/scripts/prepare_task_prompt.sh login-page T1
```

**复制输出给 AI**，AI 会：

1. 只实现 T1 的内容，不做其他改动
2. 行为变化自动补测试
3. 补注释（类/模块、方法、条件分支、风险逻辑）
4. 更新 `04-acceptance.md` 记录测试证据、风险、回滚方案
5. 输出变更文件清单和建议 commit message

**你确认**：检查代码和测试，确认无误后继续下一个 task。

### 2.4 阶段三：准备 PR

task 完成后，让 AI 生成 PR 材料。

**运行脚本生成提示词**：

```bash
bash .ai-harness/.ai-standards/scripts/prepare_pr_prompt.sh login-page T1 main feature-login-T1
```

参数说明：`<spec-id> <task-id> <目标分支> <当前特性分支>`

**复制输出给 AI**，AI 会：

1. 检查改动范围是否只覆盖 T1
2. 输出完整 PR 描述：spec link、changed files、test evidence、risks、rollback plan
3. 给出建议 commit message

**你确认**：检查 PR 材料，确认后提交推送。

### 2.5 一键串行执行（推荐给熟练成员）

如果你想让 AI 从头到尾一口气走完（需求 → spec → 一个一个 task），用串行脚本：

**运行脚本生成提示词**：

```bash
bash .ai-harness/.ai-standards/scripts/prepare_serial_task_prompt.sh login-page docs/requirements/login-page.md main
```

参数说明：`<spec-id> [需求文档路径] [目标分支]`
- 需求文档可省略（spec 已存在时直接继续）
- 目标分支可省略（默认当前分支）

**复制输出给 AI**，AI 会：

1. 先做 Startup Checks（确认仓库状态、读取规范）
2. 如果 spec 不存在，先生成 spec 四件套，暂停让你确认
3. 确认后按 `03-tasks.md` 依赖顺序逐个执行 task
4. 每个 task 完成后**自动暂停**，输出验证证据和风险，等你回复"继续"才做下一个
5. M/L 任务和高风险操作会暂停要求你确认

## 第三步：团队其他成员 clone 后怎么用

新成员 clone 了业务仓库后，只需拉取 submodule：

```bash
cd ~/workspace/my-business-repo
git submodule update --init --recursive
```

然后 `./ai status` 验证可用，即可按照第二步的流程使用脚本生成提示词。

## 底线规则（让 AI 遵守）

每次把提示词给 AI 前，这几条不需要你额外提醒，模板里已经写好了，但你需要知道：

| 规则 | 说明 |
|---|---|
| 一个 task 一个 PR | 不要把多个 task 混在一起 |
| 行为变化补测试 | bug 修复加回归测试 |
| 注释覆盖关键点 | 类/模块、方法、条件分支、边界、风险逻辑 |
| 高风险必须暂停 | DB migration、权限、计费、认证 → AI 会停下来问你 |
| 每个 task 完成后暂停 | AI 输出验证结果后等你确认才继续 |

## 排错速查

| 问题 | 解决方式 |
|---|---|
| `./ai` 报错找不到 standards | `git submodule update --init --recursive` |
| AI 没读规范 | 第一条消息加上："请先读取 .ai-harness/.ai-standards/AGENTS.md 和 standards/*.md" |
| AI 一次做了多个 task | 立即叫停，`git checkout` 回退到上一个 commit |
| 规范升级后出问题 | `cd .ai-harness/.ai-standards && git checkout <旧tag> && cd ../..` 再 `bash bootstrap_repo.sh` |
| 项目特例怎么写 | 写到 `.ai-harness/AGENTS.md`，每条带 `Reason / Scope / Expiry` |
